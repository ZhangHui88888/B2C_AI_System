# 2.1 基础 SEO/GEO

## 功能概述

实现完整的SEO优化和GEO（Generative Engine Optimization）优化，提升搜索引擎和AI引擎的可见性。

## SEO组件

### Schema标记组件

#### ProductSchema ([`ProductSchema.astro`](../../frontend/src/components/seo/ProductSchema.astro))
**用途**: 产品结构化数据标记

**输出内容**:
```json
{
  "@context": "https://schema.org/",
  "@type": "Product",
  "name": "产品名称",
  "image": ["图片URL"],
  "description": "产品描述",
  "brand": { "@type": "Brand", "name": "品牌名" },
  "offers": {
    "@type": "Offer",
    "price": "99.99",
    "priceCurrency": "USD",
    "availability": "https://schema.org/InStock"
  }
}
```

#### BreadcrumbSchema ([`BreadcrumbSchema.astro`](../../frontend/src/components/seo/BreadcrumbSchema.astro))
**用途**: 面包屑导航标记

**示例**: 首页 > 分类 > 产品

#### OrganizationSchema ([`OrganizationSchema.astro`](../../frontend/src/components/seo/OrganizationSchema.astro))
**用途**: 组织/品牌信息标记

**包含**: Logo、社交媒体、联系方式

#### FAQSchema ([`FAQSchema.astro`](../../frontend/src/components/seo/FAQSchema.astro))
**用途**: FAQ页面结构化数据

**格式**: 问答对列表

#### ArticleSchema ([`ArticleSchema.astro`](../../frontend/src/components/seo/ArticleSchema.astro))
**用途**: 博客文章标记

**包含**: 标题、作者、发布日期、图片

#### PersonSchema ([`PersonSchema.astro`](../../frontend/src/components/seo/PersonSchema.astro))
**用途**: 作者信息标记

**包含**: 姓名、简介、社交链接

### 动态生成端点

#### sitemap.xml ([`sitemap.xml.ts`](../../frontend/src/pages/sitemap.xml.ts))
**功能**: 动态生成站点地图

**包含页面**:
- 静态页面（首页、关于等）
- 产品页面（所有上架产品）
- 分类页面
- 博客文章
- 作者页面

**优化**:
- 包含lastmod、changefreq、priority
- 考虑使用KV缓存（TTL 1小时）
- 支持分片（大型站点）

#### robots.txt ([`robots.txt.ts`](../../frontend/src/pages/robots.txt.ts))
**功能**: 搜索引擎爬虫规则

**配置**:
```
User-agent: *
Allow: /

User-agent: GPTBot
Allow: /

User-agent: ClaudeBot
Allow: /

User-agent: PerplexityBot
Allow: /

Sitemap: https://yourdomain.com/sitemap.xml
```

## SEO最佳实践

### Meta标签
- **Title**: 50-60字符，包含关键词
- **Description**: 150-160字符，吸引点击
- **Canonical**: 完整URL，避免重复内容
- **Open Graph**: 社交媒体分享优化
- **Twitter Card**: Twitter分享优化

### URL策略
- 使用语义化slug
- 避免动态参数索引
- utm_*参数使用canonical指向无参数URL

### 图片优化
- 使用alt属性描述
- 使用WebP格式
- 实现懒加载
- 提供响应式图片（srcset）

## GEO最佳实践

### AI爬虫友好
- 允许GPTBot、ClaudeBot等
- 提供清晰的内容结构
- 使用语义化HTML

### 内容格式
- 使用问答格式
- 提供摘要/TL;DR
- 标注发布日期
- 署名作者信息

### 品牌可见性
- 在内容中自然提及品牌
- 提供品牌背景信息
- 展示专业性和权威性

## 性能优化

### Schema组件
- 使用内联JSON-LD，避免额外请求
- 只在需要的页面加载对应Schema
- 压缩JSON输出

### Sitemap生成
- 使用增量生成
- KV缓存（TTL 1小时）
- 避免每次全量查询

### 图片处理
- 使用Cloudflare Images自动优化
- CDN加速
- 懒加载非首屏图片
